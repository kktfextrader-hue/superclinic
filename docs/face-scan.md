# Face Scan — เอกสารระบบจดจำใบหน้า

> อ้างอิง `openFaceAdvanced(mode, context)` ใน superclinic.html

---

## โหมดทั้ง 3 (Scan1 / Scan2 / Scan3)

### Scan1 — บันทึกใบหน้าใหม่ (`mode = 'save'`)
- ใช้เมื่อผู้ป่วยยังไม่มีข้อมูลใบหน้าในระบบ
- แสดงกล้อง + 3 card จับมุม: **กลาง / ซ้าย / ขวา**
- capture อัตโนมัติเมื่อตรวจพบใบหน้าในแต่ละมุม
- กดปุ่ม **"บันทึกใหม่"** เพื่อยืนยัน (ไม่ auto-save)
- เปิดจาก: ปุ่ม "บันทึก" ในแถบค้นหา / เพิ่มผู้ป่วยใหม่ / modal ผู้ป่วยที่ยังไม่มีใบหน้า

### Scan2 — อ่านใบหน้า / ค้นหาผู้ป่วย (`mode = 'read'`)
- ใช้เมื่อต้องการค้นหาผู้ป่วยจากใบหน้า (เช่น ผู้ป่วยมาถึงเคาน์เตอร์)
- แสดงกล้อง + auto-scan เมื่อ quality ≥ 65%
- ผล: แสดงรายชื่อผู้ที่ใบหน้าตรงกัน (สูงสุด 3 คน) พร้อม **ชื่อ / HN / เบอร์โทร / %ความมั่นใจ**
- ผลค้างบนจอจนกว่าจะกดเลือก หรือกดสแกนใหม่
- ไม่มี 3-angle card (ไม่บันทึก เพียงค้นหา)
- เปิดจาก: ปุ่ม "สแกนหา" / ปุ่ม "ค้นหาด้วยใบหน้า" ทุกหน้า

### Scan3 — แก้ไข / อัพเดทใบหน้าเดิม (`mode = 'edit'`)
- ใช้เมื่อผู้ป่วยมีใบหน้าอยู่แล้ว แต่ต้องการบันทึกใหม่ (ใบหน้าเปลี่ยน, ภาพเก่าไม่ชัด)
- UI เหมือน Scan1 (3 angle cards: กลาง/ซ้าย/ขวา)
- กดปุ่ม **"อัพเดทใบหน้า"** เพื่อยืนยัน
- เปิดจาก: ปุ่ม "แก้ไข" ในแถบค้นหา / modal ผู้ป่วยที่มีใบหน้าอยู่แล้ว

## Entry Points (จุดเปิด modal)

| หน้า | ปุ่ม | Mode ที่เปิด |
|---|---|---|
| ทะเบียนผู้ป่วย (แถบค้นหา) | บันทึก | Scan1 |
| ทะเบียนผู้ป่วย (แถบค้นหา) | สแกนหา | Scan2 |
| ทะเบียนผู้ป่วย (แถบค้นหา) | แก้ไข | Scan3 |
| ภาพรวม (dashboard) | ค้นหาด้วยใบหน้า | Scan2 |
| ลงทะเบียนก่อนรักษา | ค้นหาจากใบหน้า | Scan2 |
| เพิ่มผู้ป่วยใหม่ (form) | ใบหน้า | Scan1 (context=addpatient) |
| ข้อมูลผู้ป่วย (modal) | เพิ่มใบหน้า | Scan1 (context=HN) |
| ข้อมูลผู้ป่วย (modal) | แก้ไขใบหน้า | Scan3 (context=HN) |

## Context

| context | ความหมาย |
|---|---|
| `null` | เปิดจากหน้าทั่วไป ไม่ล็อกผู้ป่วย |
| `'addpatient'` | เปิดจาก form เพิ่มผู้ป่วยใหม่ — save ลง hidden input |
| `'HN-XXXXXX'` | เปิดจาก modal ผู้ป่วยเฉพาะราย — save/update ตรงไปยัง HN นั้น |

---

## ตรรกะ Auto-scan (Scan2)

1. Detection loop ตรวจจับใบหน้าทุก frame
2. ถ้า yaw < 20° (หน้าตรง) + quality ≥ 65% + ยังไม่มีผล → trigger `_famDoSearch()`
3. `_famDoSearch()` เรียก API list patients → คำนวณ euclidean distance
4. ผลที่ distance < 0.7 → แสดงใน match list (สูงสุด 3 คน)
5. `resultShown = true` → หยุด auto-trigger จนกว่าจะ manual scan ใหม่

---

## Threshold

| ค่า | ความหมาย |
|---|---|
| dist < 0.45 | match ดี |
| dist < 0.70 | match แสดงในลิสต์ |
| conf ≥ 52% + clearMargin | highlight เป็น top match (กรอบเขียว) |
